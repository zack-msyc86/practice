# テストのデータ準備

`beforeEach` でデータを用意し、`test` では assert のみ行う。
`test` の中でデータを用意しない。

## ID は定数で決め打ち

テストで使う ID は `const` で固定値を宣言する。`let` + `beforeEach` 内の代入は使わない。
factory の自動生成 ID に依存せず、`create({ id: bandId })` のように明示的に渡す。

## 意図

テストケースが「何を検証しているか」だけに集中できる。
データ準備と検証が混ざると、テストの意図が読み取りにくくなる。
`let` は再代入可能なだけでメリットがない。固定値にして困ることもないので `const` を使う。
