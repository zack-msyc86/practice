# Turborepo のタスク設計

各パッケージは基本的に `generate`, `build`, `dev` の 3 つのタスクを持つ。

## generate

そのパッケージ自身が開発を始める前に必要な前処理。
コード生成、型生成など、ソースコードから自動的に導出されるファイルの生成。

例: GraphQL Codegen、Drizzle のスキーマ型生成

## build

そのパッケージの最終成果物を生成する。
このパッケージを依存に持つ他のパッケージが必要とするもの。

例: TypeScript のコンパイル、バンドル

## dev

`generate` と `build` をホットリロードで実行し続ける。
ソースコードの変更を検知して、generate → build の流れを自動で再実行する。
開発中はこれを起動しておけば、依存パッケージ側にも変更が即座に反映される。
